我希望做一个记账本的网站，网站主要使用 python flask + html/css/js，可以使用 bootstrap 辅助 UI 设计，要求 UI 简明现代。数据库存储使用 sqlite，在 python 内利用 sqlalchemy 与数据库通讯，使用的 orm 模型我已经在 models 提供，并适当使用 class 等 oop 特性，使得代码符合软件工程规范。网站是一个多用户网站，需要支持帐号密码注册和登录。具体的功能要点如下：

用户可以记录一笔新的交易，交易信息应至少包括：金额、收支类型（收入或支出）、交易分类、交易日期（可修改，默认为当前日期）、文本备注。

系统应提供交易分类管理功能。系统需要预设常用的分类（如餐饮、交通、购物等），同时必须允许用户自定义添加新分类、编辑现有分类名称以及删除分类。

系统需要提供一个主统计视图（仪表盘），用于直观展示某个时间周期内（默认为当月）的总收入、总支出以及最终结余。

系统需要提供图表化的统计分析功能。至少应包括：1. 按支出分类的饼图，用于展示各项支出的占比。2. 按时间（如最近6个月）的收支趋势折线图。

统计功能应支持多时间维度的筛选，必须支持按月和按年查看，并应提供按自定义日期范围查看统计数据的功能。

系统应具备预算管理功能。用户可以设置月度总预算，也可以为单个分类设置月度预算。当支出金额接近或超过预算时，系统应以某种形式提醒用户。

系统需要提供强大的交易查找和筛选功能。用户应能通过一个或多个组合条件进行查找，支持的条件包括：备注内的关键词、交易分类、时间范围、金额区间。
